## 분석 방법
1. 디렉토리 구조 파악 (ls, tree 등)
2. 주요 파일 식별 (tizen-manifest.xml, CMakeLists.txt, 소스 파일 등)
3. 각 파일/모듈의 역할 파악
4. 코드 품질 이슈 식별 (긴 함수, 불명확한 변수명, 주석 부재 등)
5. AI agent 친화성 평가

## 출력

다음 파일을 `output/01_analysis/` 폴더에 저장하세요.

**analysis.md** — 분석 문서
- 패키지 개요 (이름, 목적, Tizen 버전)
- 디렉토리 구조 (tree 형식)
- 주요 파일/모듈 목록 및 역할
- 코드 품질 이슈 목록 (심각도 포함)
- AI agent 친화성 평가 (낮음/중간/높음 + 이유)

**DONE.md** — 작업 요약
- 패키지 규모 (파일 수, 대략적인 라인 수)
- 가장 큰 문제 3가지
- 다음 단계(아키텍트)에게 전달할 핵심 정보
